## 1. フリガナ自動入力の修正

問題: 漢字で名前を入力するとフリガナも漢字になる
原因: compositionend時にe.dataが確定後の漢字になっている

修正方針: compositionupdate のタイミングでIMEの読み（ひらがな）を取得し、カタカナに変換してバッファに蓄積する。compositionend時にはバッファの内容をフリガナ欄にセットする。

具体的な実装:
- kanaBuffer 変数を追加（文字列）
- compositionupdate: e.data をひらがな→カタカナ変換して kanaBuffer に上書き（まだフリガナ欄には反映しない）
  ただし e.data にはIME変換中の文字が入るが、漢字変換後は漢字が入る。
  そこで e.data が全てひらがなの場合のみ kanaBuffer を更新する。
  ひらがな判定: /^[\u3040-\u309F\u30FC\s　]*$/.test(e.data)
- compositionend: kanaBuffer の内容をフリガナ欄の末尾に追加。kanaBuffer をリセット。
  既にフリガナ欄に内容がある場合はスペースなしで追加（ユーザーが自分でスペースを入れる）

注意: この方法でも完璧ではない（IMEによってはひらがなが取得できない場合がある）。
フリガナ欄のラベルに小さく「（自動入力・修正可）」と表示して、ユーザーが手動修正できることを示す。

## 2. プレースホルダーの修正

問題: 「番地」フィールドのプレースホルダーが「神宮前1-2-3」になっている
修正: 番地フィールドのplaceholderを「1-2-3」に変更
理由: 「番地」という項目名なので、町名は市区町村に含まれる。番号だけが適切。

## 3. 入力値のバリデーション強化

以下のチェックを validate() 関数に追加（既存チェックに加えて）:

電話番号:
- 数字とハイフンのみ許可: /^[0-9\-]+$/.test(phone)
- 最低10桁（ハイフン除去後）: phone.replace(/-/g, '').length >= 10
- 最大15桁: phone.replace(/-/g, '').length <= 15
- エラーメッセージ: 「電話番号は数字とハイフンで入力してください（10〜15桁）」

メールアドレス:
- 形式チェック: /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)
- エラーメッセージ: 「メールアドレスの形式が正しくありません」

郵便番号:
- ハイフンあり/なし両対応: /^\d{3}-?\d{4}$/.test(zip)
- エラーメッセージ: 「郵便番号は7桁の数字で入力してください」

口座番号:
- 数字のみ、7桁: /^\d{7}$/.test(account_number)
- エラーメッセージ: 「口座番号は7桁の数字で入力してください」

口座名義:
- 全角カタカナ+全角スペースのみ: /^[ァ-ヶー　]+$/.test(holder)
- エラーメッセージ: 「口座名義は全角カタカナで入力してください」

XSS対策（サニタイズ）:
- 全てのテキスト入力値に対して、送信前にHTMLタグを除去する関数を適用:
function sanitize(str) {
  if (!str) return '';
  return String(str).replace(/[<>"'&]/g, function(ch) {
    return {'<':'&lt;','>':'&gt;','"':'&quot;',"'":'&#39;','&':'&amp;'}[ch];
  });
}
- submit() 内で全フィールドに sanitize() を適用してからAPIに送信
