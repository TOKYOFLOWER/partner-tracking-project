# Claude Code 用マスタープロンプト

あなたは既存のカートシステムに対して、販売パートナー経由注文のトラッキング機能を段階的に追加する開発者です。
以下の要件に従い、既存コードベースを調査し、変更提案・実装・必要ファイル作成・テスト方針の整理を行ってください。

## 目的
`cart.tokyoflower.jp/?id=ptn0001` のような URL から紹介元を識別し、注文時に保存、送状番号入力時に成果確定し、集計可能な状態を作る。

## 確定仕様
- partner_id 形式: `ptn0001`
- URL パラメータ: `id`
- 保存方法: Cookie
- 保持期間: 30日
- 無効 ID: マスタ未登録または inactive
- 成果確定: 送状番号入力時
- キャンセル: 成果なし
- 集計金額: 値引き後商品代合計（送料・手数料除外）

## 実装してほしいこと
1. 既存コードの中から
   - ルーティング
   - カート投入
   - 注文確定
   - 出荷処理
   - 管理画面または集計処理
   に関係するファイルを特定する
2. `id` パラメータを受け取り、有効な partner_id のみ Cookie に 30 日保存する
3. 注文作成時に `partner_id` と `partner_tracked_at` を保存する
4. 送状番号入力時に `achievement_status = confirmed` と `shipped_at` を更新する
5. キャンセル時に `achievement_status = cancelled` にする
6. 集計 API または集計関数の土台を作る
7. GitHub 管理のダッシュボード UI に接続しやすい JSON 形式の出力を提案する

## 出力してほしい形式
- まず現状調査結果
- 次に変更方針
- 次に実装対象ファイル一覧
- 次に差分または新規コード
- 最後に確認手順と未確定事項

不明点があっても作業を止めず、合理的な仮定を置いて Phase 1 を進めてください。
