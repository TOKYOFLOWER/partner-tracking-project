## 1. エラー表示位置の修正

現在: エラーがページ下部（送信ボタンの上）に表示され、ページTOPにスクロールしてしまう
修正: エラー表示をページ上部（h1の直下）に移動する。スクロールもエラー表示位置に合わせる。

## 2. フリガナ自動入力

名前入力時にフリガナを自動補完する。
ライブラリ: https://cdnjs.cloudflare.com/ajax/libs/wanakana/5.3.1/wanakana.min.js は漢字→カナ変換ができないので不可。

代替案: IME入力中の読みを取得する方法を使う。
compositionstart/compositionupdate/compositionend イベントで IME の入力中テキスト（ひらがな）を取得し、カタカナに変換してフリガナ欄にセットする。

実装:
- name フィールドに compositionupdate イベントを設定
- IME入力中のテキスト（e.data）を取得
- ひらがな→カタカナ変換（charCode +96）
- フリガナ欄が空、またはIMEによる自動入力中の場合のみセット
- ユーザーが手動でフリガナを編集した場合は上書きしない（フラグ管理）

```javascript
// ひらがな→カタカナ変換
function hiraganaToKatakana(str) {
  return str.replace(/[\u3041-\u3096]/g, function(ch) {
    return String.fromCharCode(ch.charCodeAt(0) + 96);
  });
}
```

name フィールドの HTML に以下を追加:
@compositionupdate="onNameComposition($event)"
@compositionend="onNameCompositionEnd($event)"

Alpine.js のデータに追加:
kanaAutoFill: true,  // フリガナ自動入力が有効か

メソッド:
onNameComposition(e) {
  if (this.kanaAutoFill && e.data) {
    this.form.name_kana = hiraganaToKatakana(e.data);
  }
},
onNameCompositionEnd(e) {
  // 確定時にスペース区切りで蓄積
  // フリガナに反映済みの内容に追加
}

※完璧な変換は難しいので、あくまで補助。ユーザーが修正可能にする。
※フリガナ欄を手動編集したら kanaAutoFill = false にして自動入力を止める。
フリガナ欄に @focus="kanaAutoFill = false" は付けない（フォーカスしただけで止まるのはUX悪い）。
代わに @input でユーザーのキー入力を検知したら止める:
@input="if (!$event.isComposing) kanaAutoFill = false"

## 3. 郵便番号から住所自動入力

無料API: https://zipcloud.ibsnet.co.jp/api/search を使用。
CDNライブラリ不要、fetch で直接呼び出す。JSONP不要（CORSヘッダーあり）。

郵便番号フィールドの入力が7桁（ハイフンなし）または8文字（ハイフンあり）になったら自動検索:

```javascript
async function lookupZip(zip) {
  var cleanZip = zip.replace(/-/g, '');
  if (cleanZip.length !== 7) return;
  try {
    var res = await fetch('https://zipcloud.ibsnet.co.jp/api/search?zipcode=' + cleanZip);
    var data = await res.json();
    if (data.results && data.results.length > 0) {
      var r = data.results[0];
      // Alpine.js のデータを更新
      this.form.prefecture = r.address1;  // 都道府県
      this.form.city = r.address2 + r.address3;  // 市区町村+町域
    }
  } catch (e) {
    // エラーは無視（手動入力にフォールバック）
  }
}
```

郵便番号フィールドに @input="lookupZip(form.zip)" を追加。

銀行データAPI: https://bank.teraren.com/banks.json?name=みずほ のような無料APIは不安定なので、

支店名:

- プレースホルダーに「例: 銀座通支店」と表示

UIコンポーネント:
```html
<div class="relative">
  <input type="text" x-model="form.bank_name" @input="filterBanks()" @focus="showBankSuggest = true" @blur="setTimeout(() => showBankSuggest = false, 200)" placeholder="例: みずほ銀行" ...>
  <div x-show="showBankSuggest && filteredBanks.length > 0" class="absolute z-10 w-full bg-white border border-gray-200 rounded-lg shadow-lg mt-1 max-h-48 overflow-y-auto">
    <template x-for="bank in filteredBanks" :key="bank">
      <div @mousedown="form.bank_name = bank; showBankSuggest = false" class="px-3 py-2 text-sm cursor-pointer hover:bg-primary-50" x-text="bank"></div>
    </template>
  </div>
</div>
```

Alpine.js データに追加:
showBankSuggest: false,
filteredBanks: [],
majorBanks: [上記の配列],

メソッド:
filterBanks() {
  var q = this.form.bank_name;
  if (!q) { this.filteredBanks = this.majorBanks.slice(0, 10); return; }
  this.filteredBanks = this.majorBanks.filter(b => b.indexOf(q) >= 0).slice(0, 10);
}
